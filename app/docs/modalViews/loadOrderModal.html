The load order modal allows you to select an order of plugins to load. It appears automatically when you press the Start Session button on the Start View.

The load order modal cannot be closed. This is because opening it requires setting the game mode and initializing definitions in XEditLib, which currently cannot be undone.

Plugin states

  • Disabled - Plugins are colored red when one or more of their masters are unavailable to load. Plugins are considered unavailable to load if they are not found in your data directory or if they are disabled. You can move your mouse over a disabled plugin to get a tooltip specifying the masters which are unavailable to it. Disabled plugins cannot be activated.
  • Required - Plugins are colored green when an active plugin requires them as a master. You can move your mouse over a required plugin to get a tooltip specifying the active plugins which require it.

WYSIWYG

When a plugin is activated any masters it requires will automatically be activated as well. Similarly, when a plugin is deactivated, any plugins which require it will be deactivated as well. This means that the load order modal presents the exact load order that will be loaded into zEdit.

Hotkeys

  • Ctrl + Click - Multi-select individual plugins. Toggles the selection of the plugin clicked.
  • Shift + Click - Multi-select a range of plugins. Selects all plugins between the clicked plugin and the previously selected plugin.
  • ↑ - Moves the selection up one plugin. Can be used for multi-selection if Shift is held.
  • ↓ - Moves the selection down one plugin. Can be used for multi-selection if Shift is held.
  • Space - Toggles the activation of the selected plugins.
  • Ctrl + Space - Activates the selected plugins.
  • Shift + Space - Deactivates the selected plugins.
  • Ctrl + A - Selects all plugins.
  • Escape - Clears your current plugin selection.
  • Enter - Loads the active plugins. Equivalent to clicking the OK button.
