Below is a guide for creating zEdit modules.

Step 1: Prepare

Prepare a development environment for creating your zEdit module.

  • WebStorm
  • Sublime Text
  • Atom
  • Notepad++

Step 1: Design

Make a plan for what your module will do in plain English. You can spend as much or as little time on this step as you want. It is recommended to spend a decent amount of time on design, as it will help you avoid time-consuming problems during the implementation step.

Below are some tips for designing modules:

  • Use top-down design. Break the task into smaller tasks until each task can be completed in a few lines of code.
  • Keep it simple. If something seems too complicated, it probably is.

Step 2: Implementation

Below are some tips for implementing modules:

  • Don't repeat yourself. Use functions and data structures when applicable. You will produce better, easier to debug code.
  • Avoid premature optimization. Code the simplest solution, then optimize as necessary.
  • Don't be afraid to ask for help.
